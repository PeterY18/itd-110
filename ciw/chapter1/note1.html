Lesson 1 notes

  • Respnsive design means to create web pages that respond well to different screen sizes.
  • Apps are small applications made for mobile devices
  • html 4.01 Transactional allowed developers to use css
  • html 4.01 Strict required separation of presentation and content
  • html 4.01 Framset used html frames that could be placed inside each other
  • Extensible html, or XHTML, used parts of syntax from Extensible Markup Langugae (XML). Some of these include DOCTYPE, a single root element, and elements must be nested.
  • HTML 5 is the latest standard that introduced standardized video and audio elements, geolocation, and drag and drop capapbility
  • CSS makes it easy to change the style for all webpages if they import the same css file
  • Code validation is checking your code against the syntax rules. The code is validated once the errors are fixed
  • Strict syntax rules are follwed so most browsers render properly, make the webpage scallable, and make it accessible to more users.
  • Before the video element, developers had to use plugins like flash
  • API or application programming interface is a source code specification that allows different components to work together instead of relying on proprietary plugins
  • The canvas api allows on the fly drawing of pictures for video games. Does not need flash anymore

html elements

  • header is used for logos and news. It is found at the top
  • main hold everything besides the header and footer
  • nav is used to list links
  • section groups other elements
  • article adds content to supplement main content such as blogs, images, or tweets
  • aside is used as a sidebar for other content such as news feeds and ads
  • footer is at the bottom of the page or other grouping elements that includes copyright, owner, and contact info

video attritubes

  • width and height specify size
  • controls="controls" adds pause and play buttons
  • poster="blah.img" or the thumbnail
  • source elements is used to say where the video file is. src="path" is the syntax
  • type specifies the video type like mp4, WebM, or Ogg
  • text inside video elements is for when the browser doesnt support video

audio attributes

  • controls="controls" to add controls
  • source src="your_video" is used to find audio file
  • type specifeis audio type like mp3, WAV, or Ogg
  • extra text in case audio doesn't load
  • loop="loop" will loop the video
  • autoplay="autoplay" will play the video automatically when loaded
