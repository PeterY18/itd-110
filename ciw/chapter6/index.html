Events

  • abort: occurs when the loading of an image is aborted
  • blur: occurs when input focus is removed from a form element
  • click: when the user clicks on a link or form element
  • change: occurs when a user changes the value of a form field
  • error: when an error takes place while a page or image is loading
  • focus: when a user gives input or focus to a form element
  • load: when a page is loaded into the browser
  • mouseOver: whne the user moveshtemouse point over a link, image or other visible element on the a page
  • mouseOut: Occurs when the mouse pointer leaves a link, image or other visible element on a page
  • reset: Occurs when a form's Reset button is clicked
  • select: Occurs when the user selects the text in a form field
  • submit: Occurs when a form's Submit button is clicked
  • unload: Occurs when a page is unloaded from the browser (i.e., closed)

Errors

  • load-time: commonly called compiler or interpreter errors, these are typically syntax errors. load-time usually cause error alerts
  • run-time: after the script has loaded and is running, an error occurs. These are tupically caused by improper use of commands. run-time errors usually cause error alerts as well.
  • logic: mathematical errors, casting errors, errors in proper command usage or errors in the structure of the script, which result in the script running improperly. Logic erros do not cause error alerts. The script may return unexpected results or may fail to execute at all.
