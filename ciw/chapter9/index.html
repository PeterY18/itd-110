    input attributes and forms

    • data submissions with input attributes will overide submission in form
    • non html5 browsers will use form tag instead
    • the following are attributes for input
    • form: specifies where the form is. input elements go here
    • formaction: specifies where to send user input when the form is submitted. value is a url
    • formenctype: specifies how form input should be encrypted when submitted to the server
    • formmethod: specifies http protocol to use
    • formnovalidate: directs form to not validate
    • formtarget: Specifies a name or keyword that indicates where to display the response to the user after s/he has submitted the form. formtarget can say to open response page in a tab
    • list: Refers to pre-defined options contained in a datalist
    • min max: defines min and max values as an input
    • maxlength: defines maximum length of input
    • multiple: user can enter more than one input into the field
    • pattern: checks input field against a regular expression
    • placeholder: provides a hint to the user as to what input is expected
    • required: use most fill this field out

    buttons

    • button type="button": creates a clickable button
    • button type="submit": creates a submit button to send data to server for processing
    • button type="reset": creates a reset button, changes everything to default value
    • buttons can have text and images placed within its tags. input buttons can't

    user input validation

    user input validation ensures that user entered data is properly formatted and complete

    • it is best to validate on client and server side
    • inline validation occurs when each field of the form is validated as the user completes before clicking the submit button
    • inline validation helped increase form completion rate by 22%
    • inline validation helped complete forms faster by 42%
    • premature error messages fustrate users
    • persistent messages of success helps users
    • success messages should be placed outside the form
    • inlucde inline validation whenever possible
    • can't determine if information is true or false
    • before html5, developers depended on javascript plugins to achieve validation
    • required is not inline validation, it happens when the user submits the form

    pattern attribute

    • pattern attributes use regex to make sure a proper email, date, or url is entered
    • placeholder, pattern, required is a good way to ensure users enter the right data
